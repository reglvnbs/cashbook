# 测试与本地运行使用指南

`test.sh` 会将项目同步到 `build/test/` 并从副本运行代码，源码目录不会产生运行缓存、数据库或日志。

脚本优先使用 Python 3.13。系统未安装 Python 3.13 时，会输出 `WARNING` 并自动回退到 `python3`；也可以通过 `PYTHON_BIN` 指定其他 Python 可执行文件。所选 Python 版本变化时，脚本会自动重建隔离虚拟环境。

## 目录

```text
build/test/
├── app/                    项目代码副本
├── tests/                  测试代码副本
├── config.json             普通配置副本
├── .venv/                  共用虚拟环境
├── .pytest_cache/          pytest 缓存
└── data/
    ├── test/               pytest 数据和日志
    └── local/              本地服务数据和日志
```

正式 `data/`、`.env`、`.git` 和已有 `build/` 不会同步到副本。

## 参数

| 参数 | 适用模式 | 说明 |
| --- | --- | --- |
| `-h`, `--help` | 全部 | 显示命令速查 |
| `--local` | 本地服务 | 启动服务，不运行 pytest |
| `--host HOST` | 本地服务 | 监听地址，默认 `127.0.0.1` |
| `--port PORT` | 本地服务 | 监听端口，默认 `8000` |
| `--keep-data` | pytest | 保留上次测试数据 |
| `--reset-data` | 全部 | 清空当前模式的数据 |
| `--reset-env` | 全部 | 重建 `build/test/.venv` |
| `--` | 全部 | 后续参数传给 pytest 或本地服务 |

`--host` 和 `--port` 只能与 `--local` 一起使用。项目不提供调试模式。

## 运行测试

```text
# 运行全部测试，默认重建测试数据
./test.sh

# 保留上次测试数据
./test.sh --keep-data

# 重建虚拟环境
./test.sh --reset-env

# 把参数传给 pytest
./test.sh -- -q
./test.sh -- -k overview
./test.sh -- tests/test_api.py::test_create_transaction -vv
```

测试失败后，数据库、日志和缓存保留在 `build/test/`。

## 启动本地服务

```text
# 默认地址 http://127.0.0.1:8000
./test.sh --local

# 允许局域网访问并修改端口
./test.sh --local --host 0.0.0.0 --port 9000

# 清空本地服务数据后启动
./test.sh --local --reset-data
```

本地服务从 `build/test/` 副本启动，数据保存在 `build/test/data/local/`。按 `Ctrl+C` 停止服务。

## 配置与数据

| 内容 | pytest | 本地服务 |
| --- | --- | --- |
| 配置文件 | `build/test/config.json` | `build/test/config.json` |
| 数据目录 | `build/test/data/test/` | `build/test/data/local/` |
| 默认数据行为 | 每次重建 | 保留 |
| 保留或重建 | `--keep-data` | `--reset-data` |
| 密钥 | 测试专用值 | 从启动进程的环境变量读取 |

源码 `.env` 不会复制。需要使用 DeepSeek 时，在启动命令中提供环境变量：

```text
DEEPSEEK_API_KEY=实际密钥 ./test.sh --local
```

## 排查

| 问题 | 处理方式 |
| --- | --- |
| 依赖异常 | `./test.sh --reset-env` |
| 测试数据影响结果 | `./test.sh --reset-data` |
| 端口被占用 | 使用 `--local --port` 更换端口 |
| 需要查看失败现场 | 检查 `build/test/data/test/` 和对应日志 |
| 源码目录出现缓存 | 停止直接运行 Python，确认所有入口都经过 `test.sh` |

不要在项目根目录直接运行 pytest 或 Flask。脚本参数变化时，同步更新本指南和 `test.sh --help`。
