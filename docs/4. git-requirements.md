# Git 要求

## 分支

- 仓库使用 `main` 分支，项目修改直接在 `main` 上进行。
- 不为普通修改额外创建功能分支。
- 已有用户修改必须保留，不能使用破坏性命令覆盖。
- 不使用 `git reset --hard` 等可能丢失工作内容的命令。

## Commit

Commit 格式为：

```text
<type>: <description>
```

| 类型 | 用途 |
| --- | --- |
| `feat` | 新增功能 |
| `fix` | 修复问题 |
| `docs` | 修改文档 |
| `refactor` | 只调整代码结构 |
| `test` | 修改测试 |
| `style` | 不影响逻辑的格式调整 |
| `chore` | 依赖、构建和配置维护 |

- 每个 Commit 只包含一类修改，避免混入无关文件。
- 描述使用简短中文，说明修改结果，不记录执行过程。
- 设计讨论阶段允许提交尚未进入实现的文档。
- 代码验证根据当前协作阶段执行；需要验证时使用项目统一脚本。

## 禁止提交

- SQLite 数据库及其 WAL、SHM 文件。
- 日志、缓存、虚拟环境和构建输出。
- `.env`、API Key、Token、Cookie 和其他敏感信息。
- 临时调试文件、编辑器备份和操作系统生成文件。
- `build/` 和 `data/` 中除 `.gitkeep` 之外的运行内容。
