# 配置与密钥要求

## 设计原则

配置只用于部署环境或外部服务之间确实可能不同的值。项目自身的固定信息直接写在代码或设计文档中，不为了“以后可能会改”而增加配置项。

配置读取优先级为：

```text
测试配置 > 环境变量 > config.json > 程序默认值
```

## config.json

初始版本只保留以下配置：

| 分组 | 配置 |
| --- | --- |
| `server` | 监听地址、端口 |
| `database` | SQLite 数据库路径 |
| `logging` | 日志级别、输出方式、日志文件路径、单个日志文件轮转大小 |
| `transactions` | 流水列表每页显示数量 |
| `deepseek` | API 地址、模型名称、请求超时 |

AI 单次最大条数等其他参数使用程序默认值。只有确认存在实际修改需求时，才把它们加入 `config.json`。

每个可配置项继续使用统一结构：

```json
{
  "port": {
    "desc": "服务监听端口，范围 1-65535",
    "type": "int",
    "value": 8000
  }
}
```

配置错误在应用启动阶段给出明确原因，不静默使用无效值。

## 固定内容

以下内容不是运行配置：

- 应用名称固定为“账本”。
- 币种固定为人民币，页面语言固定为简体中文。
- 分类名称、编号、颜色和图标映射。
- 页面颜色、圆角、间距和文案。
- 数据库字段、接口路径和业务规则。

## AppConfig 对象

`config.py` 以 `AppConfig` 对象作为配置的唯一入口。JSON 字典只在读取文件时短暂存在，业务模块不能直接读取或传递原始配置字典。

`AppConfig` 负责：

- 创建包含程序默认值的配置对象。
- 从 `config.json` 导入并覆盖普通配置。
- 从环境变量覆盖部署配置和敏感配置。
- 把字符串转换为声明的数据类型。
- 校验必填值、范围和组合关系。
- 导出可供 Flask 使用的配置映射。
- 按统一格式导出 `config.json` 模板。

对象要求：

- `AppConfig.from_file()` 返回 `AppConfig`，不返回 `dict`。
- 未知配置项直接报错，不能被静默忽略。
- 对外暴露的值已经完成类型转换和校验。
- Route、Service 和 Repository 只读取最终配置，不解析 JSON 或环境变量。

## 环境变量与密钥

以下敏感值只通过环境变量提供：

| 变量 | 用途 |
| --- | --- |
| `SECRET_KEY` | CSRF 等请求防护 |
| `DEEPSEEK_API_KEY` | 调用 DeepSeek API |
| `CASHBOOK_API_TOKEN` | 自动新增流水接口鉴权 |

- `.env.example` 只包含变量名称、说明和占位值。
- `.env` 只用于本地开发，不提交 Git，也不复制进 Docker 镜像。
- 日志和错误响应不得输出环境变量值。
- 未配置可选密钥时，只禁用对应功能，不影响手动记账。
