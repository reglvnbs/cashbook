# 日志要求

## 输出

- 正式代码统一使用 Python `logging`，不使用 `print()` 记录运行状态。
- 日志同时支持控制台和文件输出。
- 文件日志默认保存在 `data/logs/`。
- Docker 环境中的关键日志能够通过 `docker logs` 查看。
- 日志文件按大小或日期轮转，防止单个文件无限增长。

日志格式：

```text
[2026-07-19 10:30:25][INFO][app/routes/api.py:53] 新增流水成功: id=123
```

## 等级

| 等级 | 用途 |
| --- | --- |
| `DEBUG` | 调试阶段的执行细节 |
| `INFO` | 启动成功、写入完成等正常事件 |
| `WARNING` | 外部服务异常、无效响应等可恢复问题 |
| `ERROR` | 当前操作失败 |
| `CRITICAL` | 应用可能无法继续运行的严重错误 |

## 敏感信息

日志不得记录：

- API Key、Bearer Token、Cookie 和 CSRF Token。
- `.env` 内容和完整环境变量。
- AI 记账原始输入和 DeepSeek 完整响应。
- 完整请求头、数据库内容和用户提交的完整请求体。

日志可以记录错误类型、接口名称、HTTP 状态、流水编号和必要的排查上下文。
