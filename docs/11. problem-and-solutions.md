# 开发问题与解决方案

本文简单记录账本应用开发和调试中遇到的问题，方便后续维护。

## 本地运行

| 问题 | 解决方案 |
| --- | --- |
| 系统没有 Python 3.13 | `test.sh` 输出警告并回退到系统 Python；需要指定版本时使用 `PYTHON_BIN`。 |
| 手机无法访问本地服务 | 使用 `./test.sh --local --host 0.0.0.0` 启动，再通过电脑的局域网 IP 访问。 |
| 本地 AI 记账没有密钥 | 启动时设置 `DEEPSEEK_API_KEY=实际密钥`，密钥不写入代码和 Git。 |

## 数据与业务

| 问题 | 解决方案 |
| --- | --- |
| 预算未设置和预算为 0 容易混淆 | 未设置使用空值；输入 0 表示“没有预算”，显示灰色 100% 进度条。 |
| 旧数据库不允许保存零预算 | 启动时迁移预算表约束，并保留原有预算数据。 |

## 页面与手机适配

| 问题 | 解决方案 |
| --- | --- |
| 手机端开始、结束日期重叠 | 日期筛选在手机端改为单列，并限制日期控件不能撑出容器。 |
| 记账和 AI 识别结果的输入框不齐或溢出 | 统一输入框高度，使用可收缩网格，并限制弹窗内容宽度。 |
| iPhone 日期文字位置偏上 | 单独设置日期内容的高度和行高，使文字垂直居中。 |
| iPhone 点击输入框后页面自动放大 | 手机端输入框、下拉框和文本域字号统一为至少 16px。 |
| 主屏幕应用切换页面时打开 Safari | 增加独立运行配置，并把总览、流水和预算统一放在根路径应用范围内。 |
| 月份输入和筛选操作不够自然 | 月份改为可视化选择；清空筛选改为较弱的重置操作。 |
| 网站缺少图标 | 使用账本品牌中的橙色 `¥` 生成浏览器和苹果主屏幕图标。 |

## 公网部署

| 问题 | 解决方案 |
| --- | --- |
| Mac 打包的镜像在服务器出现 `exec format error` | Mac 是 `arm64`、服务器是 `amd64`；使用 `docker buildx build --platform linux/amd64 --load` 构建，并在上传前检查镜像架构。 |
| 服务器有公网 IP，但账本只允许自己的设备访问 | 容器只绑定 `127.0.0.1:8000`，通过 Tailscale Serve 提供 Tailnet 内部入口，并保持 Funnel 关闭。 |
| 原有 Nginx 已占用公网 443，Tailscale 地址得到错误证书 | 不修改原有 Nginx，改用 Tailscale HTTPS 端口 8443。 |
| 部署账本时需要避免影响原有服务 | 部署前后检查 Nginx、x-ui/xray 和原站点响应；账本使用独立容器、环境文件和数据卷。 |
| 服务器系统版本较旧且有较多安全更新 | 不在应用部署过程中直接升级系统，先备份并另行安排系统升级窗口。 |

## Git 整理

开发过程中的连续小修复使用 `amend` 或 `squash` 合并为一个修复提交，避免提交历史过于零碎。
