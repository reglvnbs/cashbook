# 测试与本地运行要求

命令和配置说明见 [测试与本地运行使用指南](<7. testing-guide.md>)。

## 隔离原则

- `test.sh` 是运行 pytest 和本地服务的唯一入口。
- 脚本先把项目同步到 `build/test/`，再从该副本运行 pytest 或本地服务。
- 源码目录不能作为 Python 工作目录、导入目录或本地服务目录。
- Python/pytest 缓存、虚拟环境、数据库、日志、测试报告和临时文件全部保存在 `build/test/`。
- 编辑器、测试插件和其他工具不得绕过 `test.sh` 直接运行或导入源码。
- `PYTHONDONTWRITEBYTECODE` 等设置只能作为补充，不能替代副本隔离。

同步时排除 `.git`、`.env`、源码虚拟环境、正式 `data/`、已有 `build/` 和缓存文件。任何新增工具也必须把运行输出放入 `build/test/`。

## 环境与数据

| 内容 | 位置或规则 |
| --- | --- |
| 虚拟环境 | `build/test/.venv/`，依赖变化时更新 |
| pytest 数据 | `build/test/data/test/`，默认重建，可选择保留 |
| 本地服务数据 | `build/test/data/local/`，默认保留，可选择重建 |
| 失败现场 | 保留 `build/test/` 便于排查 |
| Git | `build/` 只提交 `.gitkeep` |

## 测试范围

- Route：状态码、JSON 结构、CSRF 和 Bearer Token 鉴权。
- Service：金额、分类、日期、预算和业务边界校验。
- Repository：查询、写入、事务和统计结果。
- 页面：关键内容、表单和响应式结构。
- DeepSeek 等外部服务使用 mock，不调用真实 API。

设计和文档调整不要求运行测试。需要验证代码时通过 `test.sh` 完成。
