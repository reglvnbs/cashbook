# 项目完成标准

本标准用于判断项目是否已经按照设计文档从零实现完成。仅创建目录、页面外观或接口占位不算完成。

## 交付内容

项目至少包含：

- `app/` 中的页面、Route、Service、Repository、配置、数据库、模板和静态资源。
- 带注释的 `schema.sql`，以及固定分类初始化数据。
- `config.json`、`.env.example`、生产和测试依赖文件。
- `tests/`、`pytest.ini`、`test.sh` 和测试使用指南。
- `wsgi.py`、`Dockerfile`、`pack.sh` 和离线部署指南。
- `.gitignore`、`.dockerignore` 和 `README.md`。

## 功能完成

- 总览、流水和预算三个页面在电脑和手机浏览器中均可使用。
- 总览和流水页面都能进入手动记账与 AI 记账。
- 流水新增、查询、修改和删除完整写入 SQLite。
- 流水支持关键词、日期、多个收支类型和多个分类组合筛选及分页。
- 预算支持选择月份、整页编辑和一次统一保存。
- DeepSeek 只生成草稿，用户确认后才写入数据库，多笔确认保持原子性。
- Bearer Token 可以通过单笔新增接口写入定期开销，不能获得其他操作权限。
- 未配置 DeepSeek API Key 或自动化 Token 时，仅禁用对应功能。

## 工程完成

- 实现遵守 Route → Service → Repository → Database 的调用方向。
- `AppConfig` 是配置的唯一入口，配置加载失败时阻止应用启动并说明原因。
- 金额、日期、错误响应和分页规则在所有入口中保持一致。
- 正式数据、测试数据和容器数据分别保存在规定目录或数据卷中。
- 源码目录中没有数据库、日志、虚拟环境、缓存、测试报告和临时文件。
- 仓库和镜像中没有真实密钥、用户账目或 DeepSeek 原始内容。

## 验证完成

- `./test.sh` 可以从 `build/test/` 副本完成全部测试。
- `./test.sh --local` 及其地址、端口和数据参数符合测试指南。
- Route、Service、Repository、页面结构和 DeepSeek mock 均有对应测试。
- `pack.sh` 可以生成带版本号的离线镜像文件。
- 导入镜像后健康检查成功，重建容器后原有账目仍然存在。
- 数据卷能够导出，并在另一台机器的空数据卷中恢复。

## 文档与 Git

- `README.md` 说明项目用途、配置入口、本地运行、测试和部署文档位置。
- 设计文档之间没有失效链接或互相冲突的要求。
- 实现变化同步更新相关设计、测试和部署文档。
- 所有项目修改按要求提交到 `main`，Commit 不混入运行产物或敏感信息。

以上各项全部满足后，项目才可以标记为完成。未完成项应明确记录，不能以演示数据、静态页面或未连接数据库的模拟结果代替。
