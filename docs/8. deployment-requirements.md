# 部署要求

项目采用 Docker 镜像离线部署。具体命令和操作流程见 [离线部署指南](<9. deployment-guide.md>)。

## 交付方式

- 使用 `pack.sh` 构建并导出镜像文件。
- 镜像使用明确的版本号，不使用 `latest` 作为正式部署版本。
- 部署机只需要 Docker、镜像文件和密钥配置，不需要项目源代码或 Python 环境。
- 同一个镜像文件应能用于首次部署、版本更新和故障恢复。

## Docker 镜像

- 镜像包含应用代码、普通配置和生产依赖。
- 镜像不包含 `.env`、Git 元数据、测试内容、缓存、数据库和日志。
- 容器使用非 root 用户运行应用。
- Gunicorn 作为默认启动命令，负责启动 `wsgi.py`。
- 容器内部监听 `8000` 端口，对外端口由部署时映射。
- 镜像自带 `/api/health` 健康检查。

## 配置与密钥

- 不敏感且各环境通用的配置可以写入镜像。
- `SECRET_KEY`、DeepSeek API Key 和新增流水接口 Token 通过环境变量注入。
- 密钥保存在部署机的独立环境变量文件中，不写入镜像、日志或数据库。
- 修改密钥后重新创建容器，不通过修改容器内部文件生效。

## 数据持久化

- 使用名为 `cashbook-data` 的 Docker Volume 挂载到容器 `/app/data`。
- SQLite 数据库、WAL、SHM 和文件日志都保存在该数据卷中。
- 应用首次启动时可以创建空数据库，但不得覆盖已有数据库。
- 停止、删除或更新容器不能删除数据卷中的内容。
- 镜像版本和数据相互独立，数据不能打包进镜像。

## 运行要求

- 容器异常退出或部署机重启后应自动启动。
- 应用主要用于内网访问，通过部署机地址和映射端口打开。
- 正常停止服务时，Gunicorn 应有时间完成当前请求并关闭数据库连接。
- 同一个数据卷同一时间只能由一个账本服务容器写入。

## 更新与迁移

- 更新使用新版本镜像重新创建容器，不在运行中的容器内替换代码。
- 更新前保留旧版本镜像，并备份完整数据卷。
- 数据库结构需要升级时，迁移过程不得删除已有流水、分类或预算。
- 数据转移到另一台机器时，应先停止服务，再导出完整数据卷。
- 新机器恢复数据时使用空数据卷，避免新旧数据混合。

## 日志与健康检查

- Gunicorn 的访问日志和错误日志输出到容器标准输出，可通过 `docker logs` 查看。
- 应用文件日志写入 `/app/data/logs/`，并按日志配置进行轮转。
- `/api/health` 用于判断服务和数据库是否可以正常工作，不修改业务数据。
- 健康检查失败时，容器状态应能反映异常，日志中应保留失败原因。

## 部署验收

- 浏览器能够通过部署机地址访问应用。
- `/api/health` 返回成功状态。
- 新增一笔流水后重新创建容器，流水仍然存在。
- 未提供 DeepSeek API Key 时，普通记账功能仍可使用。
- 导出的数据能够在另一台机器恢复并继续使用。
