# 账本数据库设计

这份文档用于确认数据库保存哪些信息。

## 一、设计范围

这是一个单人使用的简单账本，只需要三张表：

- 分类：保存内置的收入和支出分类，以及分类的颜色。
- 流水：保存用户确认后的收入和支出。
- 预算：保存每个月的总预算和各支出分类预算。

应用提供的修改功能也保持简单：

- 分类只能查看和使用，不提供新增、修改或删除。
- 流水支持新增、查看、修改和删除。
- 预算支持查看和修改。
- DeepSeek 只负责把自然语言转换成普通流水草稿，用户确认后仍然写入流水表，不单独保存 AI 记录。

不设计账户、转账、标签、附件、回收站、多用户、分类管理和预算结转等功能。

## 二、统一规则

- 金额使用整数“分”保存。例如 12.34 元保存为 1234，避免小数计算误差。
- 日期使用 `YYYY-MM-DD`，流水日期不能晚于今天。
- 收入流水只能选择收入分类，支出流水只能选择支出分类。
- DeepSeek 识别结果只有在用户确认后才写入数据库。
- API Key、用户输入原文和模型响应原文不写入数据库。
- 总览金额、分类占比和预算进度都根据流水实时计算，不重复保存。

## 三、分类表

一行代表一个内置分类。

| 信息 | 用来做什么 | 规则或示例 |
| --- | --- | --- |
| 编号 | 识别分类、决定显示顺序，并供前端匹配图标 | 初始化时明确写入，之后不能修改或复用 |
| 名称 | 页面显示的分类名称 | 例如“餐饮” |
| 收支类型 | 限定分类用于收入还是支出 | 只能是收入或支出 |
| 颜色 | 在图表和分类标记中辅助识别 | 使用低饱和度十六进制颜色 |

同一种收支类型下，分类名称不能重复。分类直接按照编号从小到大显示，不再保存单独的排序值。

分类编号不是临时生成的序号，而是前后端共同使用的固定标识。初始化数据库时明确写入每个分类的编号；以后即使调整分类名称，也不能修改已有编号。新增分类只能使用新的编号，不能占用或复用已有编号。

### 分类图标处理方式

数据库不保存图标代号、SVG、图片网址或 Emoji。前端直接维护“分类编号到线性 SVG 图标”的对应关系。如果遇到没有映射的分类编号，页面显示通用图标。

## 四、流水表

一行代表一笔已经确认并写入账本的收入或支出。

| 信息 | 用来做什么 | 规则或示例 |
| --- | --- | --- |
| 编号 | 数据库内部识别流水 | 自动生成 |
| 收支类型 | 区分收入和支出 | 只能是收入或支出 |
| 金额 | 记录本次发生的金额 | 大于零，单位为分 |
| 分类 | 说明收入来源或支出去向 | 必须选择已有分类 |
| 发生日期 | 记录账目实际发生日期 | 不得晚于今天 |
| 备注 | 保存简短补充说明 | 可以为空，最多 200 个字符 |
| 创建时间 | 记录流水何时写入 | 程序自动填写 |
| 修改时间 | 记录流水最后一次修改 | 程序自动更新 |

流水支持以下操作：

- 新增一笔收入或支出。
- 按日期、收支类型、分类和备注查询流水。
- 修改金额、分类、日期和备注。
- 直接删除不再需要的流水。

AI 记账确认后执行的也是普通流水新增操作，不需要在流水中增加 AI 专用字段。

## 五、预算表

一行代表某个月的一项预算。它可以是总预算，也可以是某个支出分类的预算。

| 信息 | 用来做什么 | 规则或示例 |
| --- | --- | --- |
| 编号 | 数据库内部识别预算 | 自动生成 |
| 月份 | 说明预算属于哪个月 | 使用 `YYYY-MM` |
| 支出分类 | 区分总预算和分类预算 | 为空表示总预算；有值表示该分类预算 |
| 预算金额 | 记录预算上限 | 大于零，单位为分 |
| 修改时间 | 记录预算最后一次修改 | 程序自动更新 |

预算遵守以下规则：

- 每个月最多有一条总预算。
- 每个月、每个支出分类最多有一条分类预算。
- 收入分类不能设置预算。
- 修改预算时，有对应记录就更新，没有对应记录就创建。
- 已使用金额、剩余金额和使用进度根据当月支出流水计算，不保存在预算表中。
- 分类预算之和不要求等于总预算。

## 六、查询和约束

实现 SQL 时只需要保留和当前功能有关的约束与索引：

- 流水按日期倒序查询的索引。
- 流水按收支类型和日期查询的索引。
- 流水按分类和日期查询的索引。
- 同一种收支类型下分类名称唯一。
- 分类编号固定，并按照编号从小到大展示。
- 流水金额和预算金额必须大于零。
- 流水引用的分类必须存在，分类不能被删除。
- 流水收支类型必须和分类收支类型一致。
- 每个月的总预算唯一。
- 每个月、每个支出分类的预算唯一。

## 七、页面功能边界

数据库设计对应的页面能力如下：

- 总览：只读取流水和预算，展示收入、支出、分类占比和预算进度。
- 流水：新增、列表、筛选、修改和删除。
- 预算：选择月份后修改总预算和各支出分类预算。
- 分类：只作为流水和预算的选择项，不提供管理页面。
- AI 记账：生成可修改的流水草稿，确认后写入流水表。

## 八、业务边界

### 日期与时间

- 应用以 `Asia/Shanghai` 时区判断今天，页面、接口和 AI 相对日期使用同一基准。
- 日期范围包含开始日期和结束日期，开始日期不能晚于结束日期。
- 流水日期和页面日期选择器的结束日期不能晚于今天。
- 创建时间和修改时间由程序生成，接口以带时区的 ISO 8601 格式返回。

### 金额与统计

- 输入金额最多两位小数，超过两位直接报错，不自动四舍五入。
- 接口金额使用两位小数字符串，页面金额统一显示两位小数。
- 结余等于收入减去支出，可以为负数。
- 支出分类占比等于分类支出除以总支出，保留两位小数；显示值之和允许因舍入不等于 100%。
- 没有支出时不计算占比，返回空的支出分类列表。

### 查询与分页

- 关键词匹配备注或分类名称，忽略首尾空白和英文字母大小写。
- 同时选择多个收支类型或分类时，同一筛选项内部使用“或”，不同筛选项之间使用“且”。
- 流水默认按照发生日期倒序、编号倒序排列，保证同一天的数据顺序稳定。
- 页码超出范围时返回空列表，不自动跳到最后一页。
- 修改筛选条件后，页面回到第一页。

### 预算

- 预算页的已使用金额统计所选自然月的全部支出。
- 总览中的预算摘要只在日期范围属于同一个自然月时显示，已使用金额统计当前选择的日期范围。
- 预算金额为空表示“未设置”；清空已有金额属于整页预算状态修改，不提供单独删除按钮或删除接口。
- 预算金额为零无效，未来月份允许提前设置预算。
- 同一次整页保存中的总预算和分类预算必须全部成功，否则保持原有数据不变。
