# 离线部署指南

部署流程为：在开发机打包镜像，将镜像文件复制到部署机，再导入并启动容器。部署机不需要项目源代码。

## 一、打包镜像

在能够使用 Docker 的开发机上进入项目目录：

```text
./pack.sh --tag 1.0.0
```

打包完成后生成：

```text
build/cashbook-1.0.0.tar
```

## 二、复制到部署机

将 `cashbook-1.0.0.tar` 复制到部署机。

在部署机创建 `cashbook.env`：

```text
SECRET_KEY=请替换为随机长字符串
DEEPSEEK_API_KEY=需要使用AI记账时填写
CASHBOOK_API_TOKEN=需要调用新增流水接口时填写
```

不使用的可选功能可以删除对应配置行。

## 三、导入镜像

```text
docker load --input cashbook-1.0.0.tar
docker volume create cashbook-data
```

`cashbook-data` 用于保存数据库和日志，删除或替换容器不会清除其中的数据。

## 四、启动服务

```text
docker run --detach \
  --name cashbook \
  --restart unless-stopped \
  --publish 8000:8000 \
  --env-file cashbook.env \
  --volume cashbook-data:/app/data \
  cashbook:1.0.0
```

浏览器访问：

```text
http://部署机地址:8000
```

## 五、检查

```text
curl http://127.0.0.1:8000/api/health
docker logs cashbook
```

## 六、更新版本

使用新版本号重新打包、复制并导入镜像，然后替换容器：

```text
docker stop cashbook
docker rm cashbook
```

再次执行启动命令，并将镜像版本改为新版本。继续挂载 `cashbook-data`，原有账目会被保留。

## 七、迁移数据到另一台机器

在原机器停止服务，并将整个数据卷导出：

```text
docker stop cashbook
docker run --rm \
  --user 0:0 \
  --volume cashbook-data:/data:ro \
  --volume "$PWD":/backup \
  cashbook:1.0.0 \
  tar -czf /backup/cashbook-data.tar.gz -C /data .
docker start cashbook
```

将 `cashbook-data.tar.gz` 和镜像文件复制到新机器。导入镜像后，在压缩包所在目录执行：

```text
docker volume create cashbook-data
docker run --rm \
  --user 0:0 \
  --volume cashbook-data:/data \
  --volume "$PWD":/backup:ro \
  cashbook:1.0.0 \
  tar -xzf /backup/cashbook-data.tar.gz -C /data
```

恢复完成后，按照前面的启动命令运行服务。恢复目标必须是空数据卷，避免新旧数据混在一起。
